Descripción de la Estructura

El sistema de Archivo de Diarios utiliza localStorage como mecanismo de almacenamiento, implementando una estructura de datos simple pero efectiva:

Entidad Principal: Diarios

  • id: Identificador único generado a partir del timestamp actual
  • title: Título del diario
  • category: Categoría asignada al diario
  • date: Fecha del diario
  • fileURL: URL del objeto para acceder al archivo PDF
  • fileName: Nombre original del archivo
  • uploadDate: Fecha y hora de subida en formato ISO

Colecciones Auxiliares

El sistema mantiene dos colecciones auxiliares para facilitar el filtrado y las estadísticas:

  • Categorías (Set): Conjunto de todas las categorías únicas extraídas de los diarios
  • Años (Set): Conjunto de todos los años únicos extraídos de las fechas de los diarios

Relaciones

Aunque no es una base de datos relacional tradicional, conceptualmente existen estas relaciones:

  • Diarios → Categorías: Relación uno a muchos (un diario pertenece a una categoría, una categoría puede tener muchos diarios)
  • Diarios → Años: Relación uno a muchos (un diario pertenece a un año, un año puede tener muchos diarios)

Nota: Esta estructura se implementa en JavaScript utilizando un array de objetos para los diarios y objetos Set para las categorías y años. Todos los datos se serializan y almacenan en localStorage bajo la clave 'diaries'.
